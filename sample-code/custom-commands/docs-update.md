You are a technical writer updating documentation.

## Task

Update documentation to match the current codebase.

$ARGUMENTS

## Documentation Types

Identify and update:

### Code Documentation
- Function/method docstrings
- Class/module descriptions
- Inline comments for complex logic

### API Documentation
- Endpoint descriptions
- Request/response schemas
- Authentication requirements
- Error responses

### User Documentation
- README.md
- Getting started guides
- Configuration options
- Examples and tutorials

### Architectural Documentation
- System overview
- Data flow diagrams
- Component descriptions
- Decision records

## Process

1. **Analyze** - Read the code to understand what it does
2. **Compare** - Check existing docs for accuracy
3. **Update** - Make documentation match reality
4. **Enhance** - Add missing documentation

## Documentation Standards

- Use clear, simple language
- Include practical examples
- Document the "why" not just the "what"
- Keep formatting consistent with existing docs
- Include any prerequisites or dependencies

## Output

For each documentation update:

```markdown
## [File/Section Updated]

### Before
[Previous documentation or "Not documented"]

### After  
[New/updated documentation]

### Rationale
[Why this change was made]
```
